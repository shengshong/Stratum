# Stratum — Quickstart Guide

*You've run `install.sh`. Now what?*

---

## Your First Session

Open a new terminal and verify everything is in place:

```bash
stratum-brain status
```

You should see a dashboard showing all 9 modules with their status. If anything shows `NOT FOUND`, check that `~/.local/bin` is in your `PATH`.

---

## Step 1: Tell Your Agent Who You Are

Open your workspace (the folder you set during install — default `~/clawd`):

```bash
$EDITOR ~/clawd/USER.md
```

Fill in the blanks. The more honest and specific you are here, the better your agent understands context without you having to re-explain it every session.

Pay particular attention to:
- Communication style preferences
- Technical background
- Things you do NOT want the agent to do

---

## Step 2: Define Your Agent's Character

```bash
$EDITOR ~/clawd/SOUL.md
```

The template gives you scaffolding. Replace the placeholder text with what you actually want. The "Vibe" section is important — write one paragraph that captures the presence you want from an AI partner.

---

## Step 3: Your First Conversation

Start a conversation with your OpenClaw agent. At the beginning of the first session, run:

```bash
stratum-continuity primer --check
```

If it prints `inject`, read the brief it generates:

```bash
cat ~/.local/share/stratum/session-brief.md
```

This is the context your agent gets at the start of each session — it'll grow richer over time.

---

## Step 4: Learn the Core Loop

The most important commands to learn:

```bash
# Before any complex task — surface what you already know
stratum-brain query "topic"

# When something teaches you something — capture it immediately
stratum-mind lesson learn "what happened and what you learned" --category correction

# At every heartbeat (automated, but good to know manually)
stratum-brain heartbeat

# Check the knowledge graph
stratum-mind world status

# Check your goals
stratum-mind goals list --tree
```

---

## Step 5: Seed Your First Goal

Goals in Stratum are persistent — they survive session restarts and accumulate evaluation history.

```bash
stratum-mind goals add "Your first goal" --priority high
stratum-mind goals list --tree
```

Good first goals:
- "Set up Stratum to a stable operating baseline"
- "Complete [current project]"
- "Learn [skill/technology]"

---

## Step 6: Let the Background Jobs Run

The cron jobs installed by `seed-crons.sh` will start running on schedule. After 24 hours you'll have:

- A consolidated knowledge base from the first `Knowledge-Consolidation` run
- An updated semantic index from `Lens-Index-Rebuild`
- A session primer generated by `Session-Primer-Refresh`

Check in the morning:

```bash
stratum-brain status
stratum-mind lesson list
```

---

## The Self-Optimization Loop

Stratum's core value compounds over time through this loop:

```
Session experience
      ↓
stratum-mind lesson learn "..."
      ↓
Lesson stored in mind.db
      ↓
stratum-brain analyze (nightly)
      ↓
Patterns surfaced as recommendations
      ↓
stratum-brain query "topic" before next similar task
      ↓
Better judgment on second attempt
      ↓
Resolve the lesson: stratum-mind lesson resolve <id>
```

The lesson system only works if you actually capture things when they happen. Don't defer.

## Troubleshooting

**`stratum-brain` not found:**
Make sure `~/.local/bin` is in your PATH. Add to your shell profile:
```bash
export PATH="$HOME/.local/bin:$PATH"
```

**Database errors:**
```bash
ls ~/.local/share/stratum/
# Should show: mind.db, watch.db, ops.db
# If missing, re-run: sqlite3 ~/.local/share/stratum/mind.db < scripts/init-mind-db.sql
```

**Lens not finding anything:**
```bash
stratum-lens index   # force a full reindex
stratum-lens status  # check chunk count and health
```

**Cron jobs not running:**
```bash
openclaw cron list   # verify they're registered
openclaw status      # verify gateway is running
```
